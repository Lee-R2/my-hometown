# 学习教练技能 - 示例对话

## 场景1：首次学习新概念

```
👤 用户: 我刚学了递归

🤖 学习教练: 太好了！让我们用费曼学习法来巩固这个概念。

第一步：你能用大白话解释一下"递归"是什么吗？
（假装你在教一个10岁的孩子，不要用专业术语）

👤 用户: 就是函数自己调用自己

🤖 学习教练: 很好的开始！那你能举一个生活中的例子吗？
比如有没有什么东西也是"自己套自己"的？

👤 用户: 俄罗斯套娃！打开一个里面还有一个，打开那个里面还有一个...

🤖 学习教练: 完美的类比！🎯

那递归能解决什么问题呢？你能举一个实际的应用场景吗？

👤 用户: 比如计算阶乘，5! = 5 × 4 × 3 × 2 × 1
每次都是 n × (n-1)!

🤖 学习教练: 对的！你发现了递归的关键：**把大问题拆成小问题**

那我问你一个问题：递归什么时候停下来？
（这是递归最容易出错的地方）

👤 用户: 当 n=1 的时候，1! = 1，不用再算了

🤖 学习教练: 完全正确！这就是"基准条件"或"终止条件"。

---

现在让我帮你整理成概念卡：

📚 **递归 - 概念卡**

核心定义：
> 函数直接或间接调用自身，通过将大问题拆解为小问题来求解

我的理解：
- 俄罗斯套娃：打开一个还有一个
- 阶乘计算：n! = n × (n-1)!

关键要素：
- ✅ 递归调用：自己调用自己
- ✅ 基准条件：知道什么时候停
- ✅ 问题规模缩小：每次调用更接近答案

易混淆点：
- ❌ 没有终止条件 → 无限递归 → 栈溢出
- ✅ 必须有明确的基准条件

📅 下次复习：明天（1天后）
---
✅ 概念卡已保存到 ./学习记录/20260410_递归.md
✅ 复习日程已创建

理解度自评：⭐⭐⭐⭐ (4/5)
你觉得这个评分准确吗？
```

## 场景2：复习已有概念

```
👤 用户: 帮我复习递归

🤖 学习教练: 好的！让我们检查一下你对递归的记忆。

📝 **复习模式启动**

不要看之前的笔记，先回答：

1. 什么是递归？用一句话说明。

👤 用户: 函数调用自己解决问题

🤖 学习教练: ✅ 核心概念记得清楚

2. 递归的两个关键要素是什么？

👤 用户: 嗯...一个是自己调用自己，另一个是...忘了

🤖 学习教练: 这个记得不太清楚，让我们回顾一下：

**基准条件**（终止条件）- 告诉递归什么时候停下来
**递归调用** - 自己调用自己，但问题规模要缩小

比如阶乘的例子：
- 基准条件：n=1 时返回 1
- 递归调用：n × (n-1)!

---

📊 **复习结果**：
- ✅ 核心概念：记忆清晰
- ⚠️ 关键要素：部分遗忘

建议：把"基准条件"这个词记牢，这是递归最容易出错的地方

理解度更新：⭐⭐⭐⭐ → ⭐⭐⭐⭐ (保持)
📅 下次复习：3天后

要我把更新后的概念卡保存吗？
```

## 与普通LLM的对比

### 普通LLM
```
👤 我学了递归
🤖 递归是编程中的一种技巧...（讲解概念）
（对话结束，什么都没留下）
```

### 学习教练技能
```
👤 我学了递归
🤖 让我们用费曼学习法巩固...
   → 引导你解释
   → 发现类比
   → 生成概念卡
   → 创建复习日程
   → 持久化存储
（学习轨迹被记录，后续可复习）
```

**差异化价值**：
1. 结构化输出（概念卡）
2. 学习记录持久化
3. 艾宾浩斯复习提醒
4. 学习进度追踪
