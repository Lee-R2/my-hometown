# 知识串联方法论

> 学习不是孤岛，新知识应该和你已有的认知网络连接起来。

## 为什么需要知识串联？

**认知科学告诉我们**：
- 孤立的知识点容易被遗忘
- 与已有知识建立连接，记忆更牢固
- 知识网络比知识点更有价值

**费曼学习法的核心**：
- "能用简单的话解释给外行听，才是真正理解"
- 类比、串联是深度理解的标志

## 串联的四个维度

### 1. 相似概念
"这和XX很像，但区别是..."

**示例**：
- 学习"多巴胺"时 → 串联"快乐跑步机"
- 学习"边际成本"时 → 串联"机会成本"

### 2. 对立概念
"这和XX正好相反..."

**示例**：
- 学习"wanting system"时 → 串联"liking system"
- 学习"稀缺"时 → 串联"富足心态"

### 3. 前置概念
"要理解这个，需要先知道XX..."

**示例**：
- 学习"复利"前 → 需要先理解"指数增长"
- 学习"机会成本"前 → 需要先理解"资源的稀缺性"

### 4. 应用延伸
"这让我想到你之前关注的XX问题..."

**示例**：
- 学习"多巴胺机制"时 → 延伸到主人关注的"工作倦怠问题"
- 学习"决策疲劳"时 → 延伸到"为什么晚上容易做错误决定"

## 执行步骤

### Step 1: 搜索学习记录
在生成概念卡前，先搜索：
```
- 用户之前学过什么相关主题？
- 用户的学习记录文件夹里有哪些内容？
- 用户的MEMORY.md中记录了哪些关注的问题？
```

### Step 2: 提出串联问题
在费曼提问环节增加：
- "这和你之前学过的{XX}有什么联系？"
- "你觉得这和{XX}有什么区别？"
- "这能帮助你理解之前关注的{XX问题}吗？"

### Step 3: 记录串联结果
在概念卡的"🔗 知识串联"模块中：
- 列出关联的概念
- 说明关联类型（相似/对立/前置/延伸）
- 解释为什么这个联系重要

## 串联的力量

**从"记住"到"理解"**：
- 记住：知道多巴胺是神经递质
- 理解：多巴胺是"想要"而非"喜欢"，这解释了为什么达成目标后快乐很短暂

**从"理解"到"应用"**：
- 理解：知道多巴胺的工作原理
- 应用：当发现自己在"追逐"时，意识到这是多巴胺在驱动，可以主动选择停下来

**从"应用"到"内化"**：
- 应用：用知识解决问题
- 内化：知识成为你认知网络的一部分，自然而然地影响你的思考和行为

---

**核心原则**：不要让学习成为孤岛，让每一个新知识点都找到它在你的认知网络中的位置。
