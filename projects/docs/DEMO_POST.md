## 一、Demo 简介

### 是什么

一个面向乡村小学 4-6 年级学生的**探究式 STEM 协作学习平台**（Web + PWA）。学生以 3 人小队（指引者 / 光影法师 / 秘语学者）形式，围绕"家乡"主题完成 4 阶段任务链（走进与发现 → 动手与实验 → 深入与创新 → 展示与分享），通过周期循环持续学习。平台内置两个 AI 智能体——**银蛇博士**辅导学生学习、**蜡象助手**辅助老师管理——形成"双端双智能体"协同闭环。

### 面向谁

| 用户层 | 角色 | 核心诉求 |
|--------|------|---------|
| 一级用户 | 乡村小学 4-6 年级学生 | 小队协作完成 STEM 任务，AI 教练随时辅导 |
| 二级用户 | 授课志愿者、助学老师 | 管理班级、审核产出、配置任务，AI 助手代劳行政工作 |
| 三级用户 | 学生家长（多为外出打工） | 像看朋友圈一样了解孩子学习进展，AI 对话询问情况 |

### 主要功能

**1. 小队协作 + 4 阶段任务链 + 周期循环**

- 三角色分工协作，必须齐全才能解锁主题
- 每个主题含 4 阶段任务（easy/medium/hard 三难度变体，完成任一即过关）
- 完成主题后 cycle+1，积分清零、历史归档
- sibling-teams 互斥机制（同一志愿者下的小队同周期不能选相同主题）

**【截图位置 1：小队端任务流页面 — 展示 4 阶段进度 + 三难度选择 + 当前任务卡片】**

**2. 双 AI 智能体协同闭环**

- **银蛇博士**（小队端）：学习教练 / 心理纾解 / 益智游戏 / 数据分析 / 图片视频生成；4 层记忆架构（L1 24h / L2 30 天 / L3 永久 / L4 永不），具备时间感知（注入 `[3 天前]` 时间标签）
- **蜡象助手**（管理端 + 家长端）：数据洞察 / 主题创建 / 消息推送 / 产出评价；支持语音对话（TTS/ASR）、文件上传、拖拽悬浮窗
- **跨智能体协作**：银蛇博士将小队反馈写入 `agent_communications` 表 → 蜡象助手主动读取，提醒老师关注

**【截图位置 2：左侧银蛇博士对话界面，右侧蜡象助手对话界面 — 展示跨智能体通信流程】**

**3. PWA + 移动端适配 + 完整激励生态**

- 移动端底部导航栏、骨架屏、离线兜底页、暗黑模式、44px 触控区域
- 积分系统（乐观锁防双花）+ 爱心碎片 → 爱心宝石 + 云朵市集（跨主题/跨学校流转）
- 家长扫码关注小队 → 查看进度 → 与蜡象助手对话

**【截图位置 3：管理员仪表盘 — 展示小队/主题/产出/学校的全局数据视图 + 蜡象助手悬浮窗】**

## 二、Demo 创作思路

### 灵感来源

乡村小学 STEM 教育长期面临三重困境：

1. **师资稀缺**：一名志愿者往往覆盖多所学校、上百学生，分身乏术
2. **协作学习难落地**：纸质任务单难以支持"分工—协作—反馈"的完整链路
3. **AI 教育资源错位**：通用 AI 助手不懂"小队三角色"、"4 阶段任务"、"周期循环"这些业务结构

我们走访乡村学校发现：孩子们天然有"我的家乡有什么"的探究欲，但缺少把好奇心转化为系统学习的方法。已有的教育平台多服务于城市学校，无法适配乡村"小班+志愿者走教"的场景。

### 想解决的问题

- **学生侧**：让 4-6 年级孩子在无志愿者在场时也能自主完成 STEM 探究任务，AI 教练能记住"上周我们聊了石头主题"这样的长期上下文
- **志愿者侧**：把志愿者从重复的产出审核、消息通知、任务配置中解放出来，让他们专注"教学"而非"行政"
- **家长侧**：让外出打工的家长能像看朋友圈一样，简单了解孩子的学习进展，甚至通过 AI 助手对话询问"我家孩子这周学了什么"

### 为什么做这个方向

- **场景真实**：乡村 STEM 教育不是"想象出来的痛点"，是教育公益组织每年投入大量资源仍未解的难题
- **AI 价值密度高**：两个智能体不是装饰，而是真实承载了"老师分身"和"家长陪伴"两种本应由人类完成的角色
- **技术挑战恰当**：4-6 年级学生识字能力有限、界面需极简，但底层业务逻辑（权限/任务组/周期/积分）复杂，正好适合 AI 辅助开发

### 4.4 开发心得与踩坑经验

**1. TRAE Agent 模式擅长复杂多步任务**

- 编写 `role_e2e_test.js` 时，只描述了"扫描所有角色主要功能"的需求，Agent 自动规划了 6 阶段测试流程，主动调用 5 个并行子任务研究各端功能，最后汇总为 32 项测试用例
- 安全漏洞修复时，Agent 自动按 P0→P1→P2→P3 优先级分批处理，每批完成后验证 TypeScript 编译，确保不引入新错误

**2. 长上下文对话适合跨文件重构**

- 双智能体记忆系统重构涉及 7 个文件（memory.ts / memory-distiller.ts / stream-handler.ts / system-prompts.ts / user/route.ts / admin-assistant.tsx / use-session.ts），单次对话中完成所有文件的协调修改
- buildInternalAuthHeaders 接入涉及 command-executor.ts + admin/assistant/route.ts + ai/chat/route.ts + ai/assistant/route.ts + stream-handler.ts 共 5 个文件，Agent 自动追踪所有调用链

**3. 踩坑：Next.js 16 Turbopack 热重载期间的 fetch 错误**

- 现象：开发服务器热重载时，`fetch('/api/xxx')` 偶发返回 HTML 错误页，`response.json()` 解析失败抛出 `Unexpected token '<'`
- 修复：所有 fetch 调用先检查 `response.ok` 再解析 JSON，热重载期间失败静默重试

**4. 踩坑：PostgreSQL ALTER TABLE DEFAULT 的行为差异**

- 现象：`ALTER TABLE ADD COLUMN DEFAULT 2` 会立即为现有行填入默认值（非 NULL），导致后续 `WHERE layer IS NULL` 条件不匹配
- 修复：改用 `WHERE layer = 2` 进行回填

**5. 踩坑：Supabase Service Role Key 泄露与轮换**

- 现象：硬编码的 service_role key 被提交到 GitHub 公开仓库
- 修复：在 Supabase Dashboard 轮换 key → 更新 .env.local → 删除临时脚本 → .gitignore 添加 `temp_*.ps1`

**6. 踩坑：CSRF 中间件阻断开发环境**

- 现象：添加 CSRF 防护后所有 POST 请求返回 403，因为默认只允许 localhost:3000 但开发服务器跑在 5000
- 修复：开发环境（`NODE_ENV !== 'production'`）放行所有 localhost 端口，生产环境严格校验 `ALLOWED_ORIGINS`
