# 问题处理手册

> 当人工测试发现问题后，按以下流程处理

---

## 一、问题分类与处理策略

| 问题类型 | 严重程度 | 处理策略 | 示例 |
|----------|---------|---------|------|
| 页面白屏/崩溃 | 阻塞 | **立即修复** | 页面加载后白屏 |
| 核心功能不可用 | 阻塞 | **立即修复** | 登录失败、提交作品报错 |
| 错误提示不友好 | 一般 | 记录，批量修复 | 显示英文错误码而非中文 |
| 样式/排版问题 | 轻微 | 记录，后续处理 | 按钮位置偏移 |
| 性能问题 | 轻微 | 记录，后续优化 | 页面加载慢 |

---

## 二、问题记录模板

发现问题时，请记录以下信息：

```
【问题编号】P0-1-01
【测试项】管理员登录
【操作步骤】1. 访问 /admin/login  2. 输入 admin/123456  3. 点击登录
【预期结果】跳转到 /admin/dashboard
【实际结果】页面显示 "Internal Server Error"，控制台报 500
【浏览器控制台错误】POST http://localhost:5000/api/auth/login 500
【截图/录屏】（如有）
```

## 三、常见问题排查指南

### 3.1 页面白屏

**排查步骤：**
1. 打开 F12 → Console，查看是否有红色错误
2. 如果是 `TypeError` 或 `ReferenceError`，说明前端代码有 bug
3. 如果是网络请求失败，检查 Network 面板

**常见原因：**
- API 返回了非 JSON 格式的响应（如 HTML 错误页）
- 前端解析 API 响应时未处理错误格式

**处理方式：**
- 如果是 API 返回格式问题 → 检查对应 route.ts 的错误处理
- 如果是前端解析问题 → 检查前端代码是否正确处理了 `{ error, code }` 格式

---

### 3.2 API 返回 500 错误

**排查步骤：**
1. 在 Network 面板找到 500 请求，查看 Response
2. 如果 Response 是 `{ error: "...", code: "INTERNAL" }` → 新格式，说明 catch 块正常工作
3. 如果 Response 是 `{ success: false, message: "Internal Server Error" }` → 旧格式，说明该路由未被迁移
4. 查看服务器终端日志，找到 `console.error` 输出的详细错误

**处理方式：**
- 新格式 500 → 检查具体错误原因（数据库连接？外部 API？）
- 旧格式 500 → 需要将该路由迁移到 `catchErrorResponse`

---

### 3.3 登录失败

**排查步骤：**
1. 检查 Network 面板中 `/api/auth/login` 的响应
2. 如果返回 401 → 用户名或密码错误
3. 如果返回 500 → 服务器错误，查看终端日志
4. 如果返回了 token 但页面未跳转 → 前端路由问题

**常见原因：**
- 测试数据未初始化 → 调用 `POST /api/init-test-data` 初始化
- 密码哈希不匹配 → 调用 `POST /api/migrate/all-passwords` 迁移

---

### 3.4 错误提示不友好

**排查步骤：**
1. 在 Network 面板查看 API 返回的 `code` 字段
2. 如果有 `code` 但前端显示了原始英文 → 前端未使用 `showApiError`
3. 如果没有 `code` 字段 → 后端未使用新的错误处理

**处理方式：**
- 后端缺少 code → 迁移到 `ApiErrors.xxx()` 或 `catchErrorResponse`
- 前端未解析 code → 使用 `apiFetch` 替代原始 `fetch`

---

### 3.5 权限错误（403）

**排查步骤：**
1. 确认当前登录的角色
2. 检查 API 路由的认证守卫（`requireAdmin`/`requireTeam` 等）
3. 检查权限配置表 `role_permissions`

**常见原因：**
- 角色权限未配置 → 在管理端"权限设置"中配置
- Token 过期 → 重新登录

---

## 四、紧急修复流程

当发现阻塞级问题时：

1. **记录问题**：按模板记录
2. **定位文件**：根据 API 路径找到对应的 `route.ts`
3. **检查代码**：确认是否使用了新的错误处理函数
4. **修复并验证**：
   - 修改代码
   - 保存后 Turbopack 自动热更新
   - 刷新页面验证
5. **回归测试**：确认修复未引入新问题

## 五、API 快速参考

| 错误码 | HTTP 状态 | 含义 | 前端提示建议 |
|--------|----------|------|-------------|
| VALIDATION_ERROR | 400 | 参数校验失败 | "请检查输入内容" |
| UNAUTHORIZED | 401 | 未认证 | "请先登录" |
| FORBIDDEN | 403 | 权限不足 | "没有操作权限" |
| NOT_FOUND | 404 | 资源不存在 | "请求的资源不存在" |
| CONFLICT | 409 | 数据冲突 | "数据已存在" |
| RATE_LIMITED | 429 | 请求过频 | "操作过于频繁，请稍后" |
| DB_ERROR | 500 | 数据库错误 | "操作失败，请稍后重试" |
| INTERNAL | 500 | 内部错误 | "服务器内部错误" |
| EXTERNAL_ERROR | 502 | 外部服务错误 | "服务暂时不可用" |
